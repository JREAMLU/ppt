## 为什么要熔断

-   过载保护机制 |
-   客户端请求量大 |
-   并且服务端出现大量失败或超时 |
-   防止过多错误导致服务器雪崩 |
-   保险丝 |
-   一般在客户端进行保护 |

## 熔断的三种状态

<br/>
-   闭合(Closed) |
-   半断开状态(HalfOpen) |
-   断开状态(Open) |

---

## 熔断的请求统计

<br/>
-   请求数 |
-   请求成功数 |
-   请求失败数 |
-   连续成功数 |
-   连续失败数 |

---

## 熔断的过程

-   正常访问 |
-   服务端出现一定量的失败或超时 |
-   命中条件 |
-   进入断开状态 |
-   等待时间 |
-   进入半断开状态 |
-   允许一定的流量进入 |
-   没毛病则进入闭合 |
-   有毛病则继续进入断开 |
